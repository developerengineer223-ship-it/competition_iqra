## 🔄 WORKFLOW EXAMPLES

### User Registration Flow
```
1. User fills registration form
   ├─ Enter username, email, password
   ├─ Select language preference
   └─ Click "Register"

2. Server validates input
   ├─ Check required fields
   ├─ Verify username unique
   ├─ Verify email unique
   └─ Validate format

3. Password hashing
   ├─ Use Werkzeug generate_password_hash()
   ├─ PBKDF2:SHA256 algorithm
   ├─ Automatic salt generation
   └─ Store hashed password

4. User record creation
   ├─ Add to database
   ├─ Set language preference
   ├─ Record creation timestamp
   └─ Commit transaction

5. Response to user
   ├─ Show success message (bilingual)
   ├─ Redirect to login page
   └─ Clear form
```

### User Login Flow
```
1. User fills login form
   ├─ Enter username & password
   └─ Click "Login"

2. Server validates
   ├─ Query user by username
   ├─ Verify user exists
   └─ Check password match

3. Password verification
   ├─ Use Werkzeug check_password_hash()
   ├─ Time-safe comparison
   ├─ Handle invalid password
   └─ Prevent timing attacks

4. Session creation
   ├─ User loader setup
   ├─ Session token generated
   ├─ Language preference set
   └─ Session timeout configured

5. Response to user
   ├─ Redirect to homepage
   ├─ User logged in notification
   └─ Language applied
```
